# Glossary

## A

**Active artifact**
: A saved memory currently eligible for room playback, until revoked or expired.

**Afterlife**
: The period after intake where a memory can resurface through room composition.

## B

**Bounded system**
: A system that intentionally limits scope, data collection, and feature expansion.

## C

**Consent mode**
: Intake choice that shapes retention and playback behavior (`ROOM`, `FOSSIL`, `NOSAVE`).

**Cluster readiness (`/readyz`)**
: Broad health view including worker/beat and other runtime signals.

## D

**Deployment temperament**
: Behavior profile (`memory`, `question`, `repair`, `prompt`, `witness`, `oracle`) that tunes resurfacing style.

## E

**Ephemeral artifact**
: One-time path tied to NOSAVE, revoked after single fetch.

**Export bundle**
: Portable snapshot package used for handoff or off-machine archival.

## F

**FOSSIL**
: Consent path where derivatives/residue may persist beyond raw audio retention.

## H

**Heartbeat**
: Recency signal from a process (worker, beat, optional presence sensor) used in readiness checks.

## I

**Installation profile**
: Preset tuning posture for environment-level defaults.

## K

**Kiosk (`/kiosk/`)**
: Public recording surface for intake and consent selection.

## L

**Local-first**
: Runtime posture where operation and stewardship are centered on the local node.

## M

**Memory Engine**
: Canonical system name for the full appliance.

**MinIO**
: Local S3-compatible object storage used for raw/derivative media bytes.

## N

**Node status**
: Operator synthesis endpoint used by `/ops/` to render warnings and posture.

**NOSAVE**
: Consent path for one-time playback with no long-term archive presence.

## O

**Ops (`/ops/`)**
: Steward surface for health checks, controls, warnings, and recovery actions.

## P

**Playback wear**
: Incremental scoring that affects resurfacing behavior over time.

**Public ritual**
: The participant/steward interaction sequence around recording, listening, and care.

## R

**Redis**
: Shared cache/broker backing queue signals, heartbeats, and throttling posture.

**Revocation (`/revoke/`)**
: Participant-facing flow for withdrawing a saved recording using receipt code.

**Room (`/room/`)**
: Dedicated listening surface that composes resurfacing playback.

**Room Memory**
: Participant-facing language for the memory/listening experience.

## S

**Steward**
: Operator role responsible for machine health, participant clarity, and bounded intervention.

**Surface contract**
: Explicit browser/API boundary specifying which layer owns behavior.

## T

**Task pipeline**
: Background jobs for derivatives, expiry, and cleanup.

## W

**Wear lane**
: Playback classification (`fresh`, `mid`, `worn`) used in room composition.
