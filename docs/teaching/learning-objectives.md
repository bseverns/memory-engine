# Learning Objectives

These objectives frame Memory Engine as a technical, artistic, and stewardship system.

## System Understanding

By the end, learners should be able to:
- describe Memory Engine as a local-first appliance with bounded surfaces
- explain the artifact lifecycle from ingest through wear, revocation, and expiry
- identify the role split between `/kiosk/`, `/room/`, `/ops/`, and `/revoke/`

Primary references:
- [memory-lifecycle.md](../memory-lifecycle.md)
- [how-the-stack-works.md](../how-the-stack-works.md)
- [surface-contract.md](../surface-contract.md)

## Operational Competence

Learners should be able to:
- check node posture (`ready`, `degraded`, `broken`) and interpret warnings
- run a backup/export flow and explain what each artifact contains
- perform a restore rehearsal safely and narrate risk controls

Primary references:
- [maintenance.md](../maintenance.md)
- [OPERATOR_DRILL_CARD.md](../OPERATOR_DRILL_CARD.md)
- [HANDOFF_REHEARSAL.md](../HANDOFF_REHEARSAL.md)

## Consent and Governance Literacy

Learners should be able to:
- explain differences between ROOM, FOSSIL, and NOSAVE pathways
- explain revocation and retention in participant-facing language
- identify governance boundaries the project deliberately maintains

Primary references:
- [participant-prompt-card.md](../participant-prompt-card.md)
- [ARCHIVE_STEWARDSHIP.md](../ARCHIVE_STEWARDSHIP.md)
- [MISSION_EXPANSION.md](../MISSION_EXPANSION.md)

## Experimental Evaluation Practice

Learners should be able to:
- design a bounded pilot note-taking plan without surveillance expansion
- compare deployment temperaments using observable room behavior
- produce handoff-ready evidence (install rehearsal, restore drill, steward notes)

Primary references:
- [experimental-proofs.md](../experimental-proofs.md)
- [DEPLOYMENT_BEHAVIORS.md](../DEPLOYMENT_BEHAVIORS.md)
- [roadmap.md](../roadmap.md)

## Observable Outcomes

Use these as completion indicators:
- learners can trace one memory through all major states without guessing
- learners can execute at least one ops recovery drill end-to-end
- learners can produce one pilot note set and one steward handoff record
- learners can clearly articulate ethical boundaries this project will not cross
