# Discussion Prompts

Use these prompts in seminars, steward trainings, or pilot debriefs.

## System Framing

1. What changes when we treat this as a local appliance instead of a web platform?
2. Which parts of Room Memory are technical constraints, and which are artistic choices?
3. Where does the system intentionally choose legibility over automation?

## Consent and Trust

1. Can a participant explain revocation from the receipt language alone?
2. What does FOSSIL promise, and what does it not promise?
3. Where might steward language accidentally overstate retention guarantees?

## Public Interface and Ritual

1. How does splitting `/kiosk/` and `/room/` change public behavior?
2. What is gained by keeping `/ops/` separate from participant flow?
3. When does “helpful guidance” become too directive for participants?

## Operations and Stewardship

1. Which failures are loud, and which failures are soft but dangerous?
2. What evidence shows a steward can recover the node without the original author?
3. Which warnings in `/ops/` are actionable during live operation, and which are strategic?

## Evaluation and Research Boundaries

1. What can we evaluate without collecting additional participant data?
2. Which evaluation questions require real pilot evidence rather than simulation?
3. How do we keep experimental rigor without drifting toward surveillance?

## Reflection Close

1. What should remain intentionally small in Memory Engine?
2. What is one change that would deepen the machine without widening it?
3. What should be explicitly deferred to protect project identity?
