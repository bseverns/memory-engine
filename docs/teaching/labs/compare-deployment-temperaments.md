# Lab: Compare Deployment Temperaments

## Goal

Compare how deployment grammar changes resurfacing behavior without changing core surfaces.

## What This Proves

- deployment code can shift room temperament while keeping shared architecture intact
- documented deployment claims can be tested against observed behavior
- evaluation can stay bounded without turning Memory Engine into a generalized platform

## What This Does Not Prove

- long-term social impact of each temperament
- definitive artistic preference for one mode
- generalized transfer to unrelated installations

## Required Materials

- [DEPLOYMENT_BEHAVIORS.md](../../DEPLOYMENT_BEHAVIORS.md)
- [roadmap.md](../../roadmap.md)
- two selected deployment modes
- observation template with claim/confirm/unclear columns

## Time

45-65 minutes

## Setup State

1. Choose two deployment kinds (one should be `memory` unless you have a clear reason not to).
2. Keep ingest quantity and room hardware stable between runs.
3. Record your pre-test hypotheses from documented behavior claims.

## Steps

1. Choose two deployment kinds (for example `memory` and `question`).
2. Keep ingest volume and test conditions as similar as possible.
3. Run short observation sessions for each deployment.
4. Record differences in recurrence, pacing, and perceived tone.
5. Map observations to documented behavior claims.

## Expected Observations

- some deployment differences are immediate, others require repeated listening
- metadata quality can influence how strong deployment differences feel
- claim mapping reveals which behavior statements are testable in short sessions

## Common Failure Points

- comparing runs with very different ingest pools
- undocumented config changes between runs
- treating one short run as conclusive proof of a broad claim

## Instructor Notes

- ask teams to cite specific lines from `DEPLOYMENT_BEHAVIORS.md` for each claim
- require a confidence rating per conclusion
- treat contradictions as valuable evidence, not lab failure

## Optional Extension

- run the same comparison with a second cohort and compare agreement rates
- add a steward-facing interpretation pass based only on `/ops/` summaries

## Student Deliverable Format

- side-by-side temperament comparison sheet
- list of confirmed, unclear, and contradicted behavior claims

## Reflection Prompts

- Which claims are proven by this small test?
- Which require longer or live-site pilots?
