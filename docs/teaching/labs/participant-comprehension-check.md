# Lab: Run A Small Participant-Comprehension Check

## Goal

Measure whether people can understand consent and revocation with existing public language.

## What This Proves

- whether current participant-facing language supports basic informed participation
- which concepts are routinely misunderstood in first contact
- whether steward explanation scripts need revision before public pilot

## What This Does Not Prove

- full public deployment safety
- legal sufficiency for all contexts
- long-term participant trust after repeated encounters

## Required Materials

- [participant-prompt-card.md](../../participant-prompt-card.md)
- [FIRST_DAY_PACKET.md](../../FIRST_DAY_PACKET.md)
- [templates/participant-comprehension-template.md](../templates/participant-comprehension-template.md)
- small internal test cohort (not public participants)
- consistent set of 3-5 comprehension questions

## Time

30-45 minutes

## Setup State

1. Confirm this is an internal rehearsal, not public data collection.
2. Prepare neutral facilitation script with minimal prompting.
3. Decide how responses will be scored before sessions begin.

## Steps

1. Recruit a small internal cohort (classmates/staff), not public participants.
2. Let each person complete one kiosk flow with minimal assistance.
3. Ask 3-5 comprehension questions immediately after receipt handoff.
4. Score responses using the template.
5. Identify copy or ritual changes needed before public pilot.

## Expected Observations

- participants usually understand some parts of consent quickly and miss others
- revocation pathway comprehension varies with wording clarity
- repeated confusion patterns point to copy fixes, not participant failure

## Common Failure Points

- facilitator giving extra hints that mask true comprehension gaps
- changing questions between participants
- collecting too little evidence to identify patterns

## Instructor Notes

- keep tone non-evaluative; participants are testing language, not being graded
- capture exact confusing phrases where possible
- if comprehension is weak, revise copy before scaling pilot scope

## Optional Extension

- compare comprehension with and without a steward verbal explanation
- run a second pass after copy edits and measure change

## Student Deliverable Format

- completed comprehension template
- misunderstanding clusters
- proposed wording or steward-script adjustments

## Reflection Prompts

- What did people reliably understand?
- What remained ambiguous even after one pass?
